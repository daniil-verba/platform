# 🤝 Как внести вклад в Hivemind

Спасибо, что хотите помочь проекту! Мы рады любым вкладам — от исправления опечаток до новых фич.

## 🐛 Сообщение об ошибке

1. Проверьте, не существует ли уже [открытого issue](https://github.com/daniil-verba/hivemind/issues).
2. Создайте новый issue с подробным описанием:
   - Версия ОС и компилятора.
   - Шаги для воспроизведения.
   - Ожидаемое и фактическое поведение.
   - Если возможно, приложите минимальный пример кода.

## 💡 Предложение новой фичи

1. Создайте issue с пометкой `feature request`.
2. Опишите проблему, которую решает фича.
3. Опишите предполагаемый интерфейс/API.

## 🔧 Разработка

### Требования
- CMake 3.15+
- Компилятор с C++20
- Google Test для тестов

### Сборка для разработки
```bash
git clone https://github.com/daniil-verba/hivemind.git
cd hivemind
mkdir build && cd build
cmake -DBUILD_TESTS=ON ..
make -j$(nproc)
ctest --output-on-failure
```

### Стиль кода
- Используем `clang-format` с конфигурацией из корня проекта.
- Имена: `PascalCase` для классов, `snake_case` для функций.
- Комментарии к публичному API на русском и английском.

### Процесс Pull Request
1. Форкните репозиторий.
2. Создайте ветку: `feature/your-feature-name` или `fix/bug-name`.
3. Внесите изменения.
4. Напишите тесты для новых фич.
5. Убедитесь, что все тесты проходят.
6. Создайте Pull Request в ветку `main`.
7. Опишите изменения в PR.

## 📄 Лицензия

Внося изменения в проект, вы соглашаетесь передать права на них под той же лицензией (AGPL v3), что и сам проект.
